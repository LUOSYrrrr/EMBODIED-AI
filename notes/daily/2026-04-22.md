## 踩坑 & 解决（11 个，密度爆表）

### 坑 1：相机默认 YUYV@10fps，教程里的 fps=30 直接报错

`find_cameras` 默认显示 YUYV 640×480 → 最高 10fps。直接用 `fps=30` 会报 `failed to set fps=30 (actual_fps=10.0)`。

**解决**：在 camera config 里加 `fourcc: MJPG`，压缩模式 USB 2.0 带宽够跑 30fps。

### 坑 2：`/dev/ttyACM*` 权限 Denied（dialout 组）

Ubuntu 默认用户不在 `dialout` 组，访问串口直接 403。

**解决**：`sudo usermod -aG dialout $USER` + **彻底注销重登**（不是 reboot 也行，但不能只重开终端）。临时救急用 `newgrp dialout`，但会掉 conda 环境。

### 坑 3：elbow_flex (motor id=3) 总线掉包

`ConnectionError: There is no status packet!` on motor id=3。

**解决**：follower 彻底断电重启（先拔电+USB，等 5s，先插电再插 USB）。上次 teleop 崩溃会把总线留在半 torque 状态。如果断电不解决 → 物理检查 elbow_flex 前后两段 TTL daisy chain。

### 坑 4：`save_episode` 报 "add frames before add_episode"

按 → 太快，在 `record_loop` 采到第一帧前就触发 `exit_early` → episode 0 帧 → save 崩溃。

**解决**：听到 `Recording episode N` 后**等至少 3 秒再操作**，做完动作再按 → 结束。Reset 阶段同理，让 10s 自然走完不要按 →。

### 坑 5：误以为数据全丢了（50→5）

pandas 只读 `meta/episodes/chunk-000/file-000.parquet` 显示 5 条，实际 `info.json` 的 `total_episodes: 44`。

**真相**：新版 LeRobot 把**多个 episode 打包到一个 parquet/mp4**。`meta/episodes/chunk-000/` 下有 6 个 `file-0XX.parquet`，每个装几条。查完整数量要 `concat` 所有文件。

### 坑 6：`--resume=true` 强制要显式 `--dataset.root`

不给 root 会报：`resume() requires an explicit 'root' directory`。

**解决**：`--dataset.root=$HOME/.cache/huggingface/lerobot/<user>/<dataset>`。

### 坑 7：`~` 在 `--root=~/...` 不展开

bash 只在 `~` 位于 word 开头时展开，`--root=~/foo` 属于等号后的，**不展开**。LeRobot 拿到字面 `~` 就去 HF 查远端，401 Unauthorized。

**解决**：用 `$HOME` 或绝对路径。

### 坑 8：LeRobot CLI 参数命名不统一

- `lerobot-record` / `train` / `teleoperate`：`--dataset.repo_id`（下划线 + dot）
- `lerobot-edit-dataset`：`--repo_id`（下划线）
- `lerobot-dataset-viz`：`--repo-id`（连字符） ← **唯一**

已记到 [so101_hardware.md](../so101_hardware.md)。

### 坑 9：LeRobot extras 要按需补装

每次报 `'xxx' is required but not installed` 都要从源码目录补装，**不能用 PyPI**：

```bash
cd ~/embodied-ai/lerobot
pip install -e ".[viz]"        # rerun-sdk
pip install -e ".[dataset]"    # datasets
pip install -e ".[training]"   # accelerate + wandb
pip install pynput             # 键盘监听，[dataset] 没带
```

PyPI 里的 `pip install lerobot[xxx]` 会覆盖掉 editable 安装，必坑。

### 坑 10：HF username ≠ git username

git: `LUOSYrrrr`（4 r）；HF: `LUOSYrrrrr`（**5 r + 大写 LUOSY**）。写错会 `403 Forbidden: You don't have the rights to create a dataset under the namespace`。

**查法**：`hf auth whoami`。

### 坑 11：`ds.push_to_hub(repo_id=...)` 参数没生效

`LeRobotDataset.push_to_hub()` 用的是 `self.repo_id`，**不接受 `repo_id` 作参数**（会被吞进 `**card_kwargs`）。

**解决**：push 前手动赋值 `ds.repo_id = 'LUOSYrrrrr/so101_pickplace_v1'`。

## 教训

1. **快速迭代框架的 CLI / API 不稳定** — LeRobot 这种活跃项目，参数命名、meta 格式、extras 划分都在变。踩坑率远高于"看教程"的心理预期。
2. **真机数据采集的失败成本**在"误按键"而不是"动作不完美"— 按 → 太快比抓歪更致命。
3. **新版格式必须先读 `info.json`**，别拿 pandas 数 parquet 的行数当真（打包分片会骗你）。
4. **transfer 硬件状态不能 transfer calibration 直觉** — Mac 标过不等于 Linux 能直接跑，注销重登、dialout 组、串口号都得重过一遍。
