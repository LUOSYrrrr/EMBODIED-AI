## 踩坑 & 解决

### 坑 1：Isaac Sim 启动报 ROS2 Bridge startup failed

```
[Error] [isaacsim.ros2.bridge.scripts.extension] ROS2 Bridge startup failed
[ext: isaacsim.ros2.bridge-4.1.15] shutdown
```

**原因**：Isaac Sim 4.5 默认加载 ROS 2 Bridge 扩展，需要 Humble 的 `rclcpp` / FastDDS 等依赖。但：
- 系统 `ROS_DISTRO` 为空，`/opt/ros` 目录不存在
- ROS 2 Humble 按 plan 是 Week 3 Day 15（05-04）才装

**结论**：**非阻塞**。Week 1-2 的 Franka RL 训练完全不用 ROS 2 Bridge。

**处理**：
- 方案 A（推荐）：忽略，继续做主线
- 方案 B（想日志干净）：`isaacsim --/exts/isaacsim.ros2.bridge/enabled=false`
- 方案 C：提前装 ROS 2 Humble（不推荐，打乱节奏）

### 坑 2：IsaacLab `main` 分支 ≠ Isaac Sim 4.5 可用版本 ⚠️

这是个关键坑，差点直接 `./isaaclab.sh --install` 就炸。

**现状**：
- 当前分支：`main`
- `main` 最新 commit 已经是 **v3.0.0-beta**
- v3.0.0 用的是 **Newton 物理引擎**（不是 PhysX），要求 **Isaac Sim 5.x**
- 我本地装的是 **Isaac Sim 4.5.0** → 跟 `main` 不兼容

**版本对应**：

| IsaacLab | Isaac Sim | 物理 |
|---|---|---|
| v2.x（v2.3.2 最新稳定） | 4.2 / 4.5 | PhysX |
| v3.0.0-beta（main） | 5.x | Newton |

**正确操作**：

```bash
cd ~/embodied-ai/isaaclab/IsaacLab
git checkout v2.3.2
./isaaclab.sh --install
```

**教训**：clone GitHub 上的框架项目，**永远先看 tag**，不要默认用 `main`。Isaac Sim / Isaac Lab / LeRobot 这种快速迭代的项目，`main` 经常是下一个大版本的开发分支。
